# 📋 작업 브리프

**원 명령:** 결과물을 제시해봐, 빈폴더만 확인되

## 요약
내부 시스템 자동화(AEP/CIM)는 완료되었으나 사용자에게 최종 결과물이 '빈 폴더' 형태로만 노출되는 문제를 해결해야 합니다. 복잡한 기술적 성과를 비전문가도 이해할 수 있는, 구조적이고 권위적인 보고서 형태의 인터페이스로 재구성하는 것이 목표입니다.

## 분배
- **🔍 Researcher**: 지난 로그(AEP/CIM)와 회사 공동 목표를 종합 분석하여, 현재 시스템이 성공적으로 완료한 '핵심 성과 지표(KPI)' 목록과 이 데이터를 증명할 수 있는 필수적인 근거 데이터셋 3가지를 재정의하고 요약하라. (빈 폴더가 아닌, 반드시 보여줘야 할 핵심 아웃풋 명세)
- **💰 Business**: 기술적 성과(AEP/CIM 구현)를 비즈니스 가치로 전환하는 전략을 수립하라. 단순히 '작동한다'가 아니라, '운영 효율성 개선'이라는 관점에서 사용자에게 이 결과물이 제공하는 구체적인 혜택(ROI 또는 시간 절약 효과)을 KPI 기반으로 명확하게 정의하고 보고서의 스토리라인 구조를 설계하라.
- **🎨 Designer**: 빈 폴더 대신 사용할 '결과물 시각화 대시보드'의 UI/UX 디자인 브리프를 작성하라. 전문성, 신뢰도, 권위성을 강조하며 (Pretendard 폰트 기반), 핵심 KPI와 근거 데이터셋을 계층적으로 보여줄 수 있는 모듈형 레이아웃(Dashboard Template) 스케치를 제공하고, 사용자가 가장 먼저 봐야 할 '요약 및 결론' 영역의 비주얼 가이드를 확립하라.
- **💻 Developer**: 설계된 대시보드 템플릿(Designer)을 구현하기 위한 최종 프런트엔드 컴포넌트를 구축하고, Researcher가 정의한 KPI 데이터를 백엔드에서 실시간으로 불러와 노출하는 API 엔드포인트(`GET /results/visualization`)를 즉시 개발하라. 이 과정에서 로딩 시 '데이터 분석 중...' 같은 명확한 피드백 메시지를 포함시켜야 한다.
- **📱 Secretary**: 위의 모든 과정을 종합하여, 사용자에게 전달할 최종 브리핑 문서를 작성하라. 이 보고서는 1) 문제 제기(빈 폴더), 2) 원인 설명(기술적 완성도에 따른 임시 상태), 3) 해결책 제시(새로운 대시보드 접근 방식 안내)의 흐름으로 구성되어야 하며, 사용자에게 다음 액션 아이템을 명확하게 지정해야 한다.
