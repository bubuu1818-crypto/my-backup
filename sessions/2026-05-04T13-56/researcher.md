### 🎯 I. 핵심 성과 지표 (KPI) 재정의 및 측정 로직

기존の 조회수/도달률 중심의 KPI는 폐기하고, 회사가 목표로 하는 **'운영 프로세스 종속성 확보(Operational Dependency)'**에 초점을 맞춘 다음 3가지 KPI를 표준으로 채택합니다.

| No. | 핵심 성과 지표 (KPI) | 측정 목적 및 학술적 근거 | 계산/측정 방법론 | 목표치 예시 (Benchmark) |
| :---: | :--- | :--- | :--- | :--- |
| **1** | **Pain Point 전환율 (PCI, Pain Point Conversion Index)** | 단순 정보 소비를 넘어, 콘텐츠에서 제기된 '고통(Pain)'을 해결하기 위해 *실제 상담/문의*로 연결되는 비율. 신뢰 기반의 서비스 전환력을 측정합니다. | $\text{PCI} = \frac{\text{콘텐츠별 최종 문의 건수}}{\text{해당 콘텐츠 노출 총 사용자 수}} \times 100$ (단위: %) | $P_A$ 주제군 평균 $3.5\%$ 이상 유지 |
| **2** | **프로세스 통합 완성도 (PIC, Process Integration Coverage)** | 회사가 제안하는 솔루션(보고서, 위젯 등)이 고객사의 실제 운영 프로세스에 얼마나 필수적으로 삽입되어 사용되고 있는가. 시스템적 종속성을 측정합니다. | $\text{PIC} = \frac{\text{시스템 도입 핵심 기능 개수}}{\text{최대 가능 통합 기능 총합}} \times 100$ (단위: %) | $90\%$ 이상 달성 및 모니터링 자동화 |
| **3** | **권위적 콘텐츠 활용 지표 (ACI, Authority Content Index)** | 회사가 제공하는 데이터/분석 자료(보고서, API 등)를 활용하여 고객사 내부 의사결정 과정에 *실제로 인용*되는 빈도. 단순 조회수를 넘어 '전문성 기반 신뢰'를 측정합니다. | $\text{ACI} = \frac{\text{데이터 출처 인용 건수}}{\text{총 보고서/자료 배포 횟수}}$ (단위: Ratio) | $1$건 이상의 내부 의사결정 근거 자료로 활용 확인 |

### 💾 II. 필수 근거 데이터셋 명세 (Mock API Source Definition)

KPI의 신뢰도를 극대화하고 백엔드 레벨에서 분석 로직을 구현하기 위해, 다음 세 가지 구조화된 데이터셋을 Mock API에 적재할 것을 지시합니다. 이들은 단순히 데이터를 보여주는 것이 아니라, **분석 가능한 변수(Variable)**를 제공해야 합니다.

#### 1. [시장 트렌드 & 비교] Industry Trend Benchmarking Data (JSON/CSV)
*   **목적:** KPI-3 (ACI)의 학술적 근거를 마련하고, 경쟁 우위 분석 자료로 활용합니다.
*   **구조화 항목:** 최소 5개 이상의 산업 카테고리별 데이터를 포함해야 하며, 시간 흐름에 따른 변화(YoY/QoQ)가 필수입니다.
    *   `Category_ID`: (예: AI 도입률-병원 진료 기록 분석)
    *   `Year/Quarter`: (분석 기준 시점)
    *   `Metric_Name`: (측정 지표명, 예: 성장률, 채택율)
    *   `Value_2024Q1`: (구체적인 수치 값)
    *   `Source_Authority`: (데이터 출처 기관 및 근거 레벨 명시)

#### 2. [사용자 행동 분석] Pain Point Funnel & Drop-off Data (CSV)
*   **목적:** KPI-1 (PCI)을 극대화하고, 사용자 여정의 비효율적인 지점을 찾아내는 데 사용됩니다.
*   **구조화 항목:** 콘텐츠 소비부터 최종 문의까지의 모든 단계(Funnel Step)와 각 단계에서의 이탈 원인/지표를 포함해야 합니다.
    *   `Content_Topic`: (콘텐츠 주제 분류, 예: 만성질환 관리)
    *   `Step_Index`: (단계 번호, 1부터 순차적)
    *   `Interaction_Type`: (상호작용 유형, 예: 영상 시청 완료 / 리포트 다운로드 클릭)
    *   `Dropoff_Rate`: (해당 단계에서 이탈한 사용자 비율 %)
    *   `Pain_Point_Keyword`: (이탈 직전 사용자가 검색/언급한 핵심 키워드)

#### 3. [운영 프로세스] System Integration Status Log Data (JSON)
*   **목적:** KPI-2 (PIC)의 실시간 측정 및 개선 사항 추적에 활용됩니다.
*   **구조화 항목:** 고객사 도입 환경과 시스템이 연결된 핵심 모듈별 상태와 가치 기여도를 기록합니다.
    *   `Client_ID`: (고객사 식별자)
    *   `Module_Name`: (통합 모듈명, 예: 자동 진단 보고서 생성기)
    *   `Integration_Status`: (연결 상태, 예: Live / 베타 테스트 중 / 미구축)
    *   `Value_Add_Score`: (이 모듈 덕분에 얻은 정량적 가치 점수 - 예: 시간 절약 $h$/비용 감소 $\$$)
    *   `Last_Update_Date`: (최근 업데이트 일자)
