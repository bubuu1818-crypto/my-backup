# 📄 [UI/UX 디자인 브리프] 운영 효율성 개선 대시보드 (Operational Efficiency Dashboard)

**목적:** 단순히 데이터를 보여주는 것이 아니라, 데이터 기반으로 도출된 **'운영 프로세스 종속성(Operational Dependency)'**과 **'문제 해결의 필요성(Pain Point)'**을 시각적으로 강력하게 주입하여 사용자에게 권위적인 통찰력과 다음 액션 플랜에 대한 확신을 제공하는 것.

**핵심 원칙:**
1.  **Top-Down Authority Flow:** 결론 $\rightarrow$ 근거 데이터 $\rightarrow$ 문제 진단 순서로 정보 계층화 (가장 중요한 요약이 최상단에 위치).
2.  **Systemic Thinking:** 모든 시각화는 독립적인 차트가 아닌, 전체 운영 시스템의 일부임을 인지하게 한다.
3.  **Pretendard 기반 모듈형 설계:** 어떤 종류의 데이터(KPI, 트렌드 비교, 프로세스 흐름)가 들어와도 통일감 있는 템플릿을 유지한다.

---

## I. 대시보드 레이아웃 구조 (The Modular Skeleton)

대시보드는 총 3개의 가상 섹션으로 구성되며, 각 섹션은 독립적인 '컨설팅 모듈'처럼 기능해야 합니다.

| No. | 섹션 명칭 | 역할 및 목적 | 포함되어야 할 주요 요소 |
| :---: | :--- | :--- | :--- |
| **Zone 1** | **Executive Summary (최상단)** | **[가장 중요] 결론 제시.** 사용자가 이 보고서를 통해 *반드시* 얻어야 할 '핵심 통찰'과 '지금 당장의 액션 플랜'을 한눈에 요약하여 강제적으로 인지시킨다. | KPI 3종 요약, 핵심 위험 지표(Alert), 최종 권고 사항 (Action Recommendation) |
| **Zone 2** | **KPI & 진단 분석 (중앙)** | **[신뢰성 강화] 왜 이 결론이 나왔는지 근거 제시.** 정의된 3가지 KPI를 중심으로 운영상의 문제점을 진단하고, 수치적 증거로 뒷받침한다. | PCI/PIC/ACI 시계열 변화 차트, 병목 구간 분석 Heatmap, 비교 트렌드 점수화 (Scorecard) |
| **Zone 3** | **근거 데이터셋 & 프로세스 (하단)** | **[권위성 확보] 시스템적 깊이 제공.** 단순한 통계를 넘어, 데이터를 생성하는 원천(Mock API Data Source)와 운영 프로세스 흐름을 시각화하여 전문성을 극대화한다. | 비교 분석 차트 (Radar/Spider Chart), 데이터 Flow Diagram, 시장 트렌드 매칭 테이블 |

## II. 핵심 영역 디자인 스펙 상세 가이드라인

### 1. Zone 1: Executive Summary 비주얼 가이드 (가장 중요한 부분)

*   **레이아웃:** 대시보드의 상단 20%를 차지하며, 배경색은 매우 미니멀하고 중립적인 회백색(White/Light Gray)을 사용한다.
*   **제목 구성:** 단순히 '요약'이 아닌, **[핵심 메시지] + (측정 지표) = [의사결정 결과]** 구조로 작성되어야 합니다.
    *   *(예시: "PCI 3.5% 달성을 위해 A 프로세스 개선 필요")*
*   **구성 요소:**
    *   **Headline Scorecard (최상단):** 가장 중요한 KPI 3종(PCI, PIC, ACI)을 큰 숫자와 함께 배치하고, 전월/전분기 대비 변화율 $(\uparrow / \downarrow)$과 색상 코드를 명확히 표시한다.
        *   *색상 규칙:* 목표치 하회 시 (🔴 Red), 경고 단계 시 (🟠 Orange), 정상 작동 시 (🟢 Green).
    *   **Critical Alert Box:** 현재 운영에 가장 치명적인 문제(Pain Point) 1~2가지를 별도의 박스 안에 빨간색 테두리나 배경으로 강조하여 강제적으로 눈에 띄게 한다.

### 2. KPI 및 데이터 시각화 스펙 (Zone 2 & 3 공통)

*   **폰트:** Pretendard (모든 제목과 본문).
    *   Headline (섹션 제목): Bold, Size 18px~24px
    *   Body Copy: Regular/Medium, Size 12px~16px
    *   Metric Value (핵심 숫자): ExtraBold, Size 36px 이상 (가장 크게)
*   **컬러 팔레트:**
    *   **Primary (신뢰/권위):** `#0A2E5B` (딥 네이비 또는 다크 블루 계열). 모든 타이틀, 경계선, 핵심 액션 버튼에 사용.
    *   **Secondary (강조/주의):** `#D9534F` (진한 레드). KPI가 목표치 이하일 때, 혹은 '위험'을 알릴 때 사용.
    *   **Tertiary (긍정/성장):** `#5CB85C` (신호 녹색). KPI가 목표치를 초과하거나 개선되었음을 나타낼 때 사용.
    *   **Background:** `#F9FAFB` (매우 연한 회색). 눈의 피로도를 낮추고 모듈 경계를 명확히 한다.

### 3. 인터랙션 및 사용자 경험(UX) 가이드라인

*   **호버 효과:** 모든 차트와 데이터 블록은 마우스 오버 시 미세한 애니메이션(Opacity 변화 또는 그림자 추가)을 주어 '클릭 가능한 정보'라는 느낌을 부여한다.
*   **정보 확장:** KPI 수치를 클릭하면, 해당 지표의 **'상세 분석 보고서 (Deep Dive)'** 페이지로 연결되는 팝업이나 새 섹션이 열리도록 설계하여 깊이를 확보한다.
*   **레이블링:** 모든 차트에는 X축과 Y축 레이블 외에, 데이터가 의미하는 바(예: "전월 대비 감소")를 주석으로 반드시 달아준다.
