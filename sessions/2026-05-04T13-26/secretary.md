# 📑 [공지 및 표준화] Pretendard 전사적 활용 가이드라인 확립 보고서 v1.0
**(발신: Secretary Office | 목적: 디자인 통일성 확보 및 콘텐츠 제작 효율 극대화)**

CEO 지시에 따라, 새로운 표준 서체(Pretendard)의 전사적 적용을 위한 공지사항 초안과 실무 팀원용 체크리스트를 아래와 같이 작성했습니다. 이 문서는 다음 주 브리핑에서 공식 배포되어야 할 '최종 디자인 스펙' 자료입니다.

---

### 🎯 I. 핵심 목표 및 원칙 재확인 (The Mandate)

1.  **표준 서체 의무화:** 모든 형태의 아웃풋(웹사이트, 보고서 PDF, 내부 공지사항, 프레젠테이션 등)은 **Pretendard**를 기본 서체로 채택하고 사용합니다.
2.  **가독성 기반 계층 구조:** 단순한 폰트 적용을 넘어, Designer가 정의한 타이포그래피 스케일(H1~Caption)에 따라 정보의 중요도와 위계를 시각적으로 명확히 구분하는 것을 최우선 원칙으로 합니다.
3.  **적용 범위:** 모든 콘텐츠 제작자는 본 가이드를 참고하여 작업하고, 예외 사항 발생 시 반드시 Secretary를 통해 승인을 받아야 합니다.

### ✅ II. 실무팀원용 Pretendard 체크리스트 (Actionable Checklist)

다음은 팀별/작업 유형별로 필수적으로 점검해야 할 항목들입니다. 콘텐츠 제작 전 이 목록을 **필수 검토**하도록 프로세스를 업데이트합니다.

| 구분 | 작업 단계 | 점검 항목 (Checklist) | Yes/No | 담당자 확인 |
| :--- | :--- | :--- | :--- | :--- |
| **📝 기획/작문 파트 (Writer)** | 1단계: 초안 작성 시 | [ ] 모든 제목(H2, H3 등)에 Pretendard Weight 및 크기 지침 준수 여부 확인. <br> [ ] 본문 텍스트는 'Body Standard' (16px, Line Height 1.7)를 기준으로 작성되었는지 검토. <br> [ ] 리포트의 결론/요약 부분은 Body Large (18~20px) 활용이 적절한지 점검. | | |
| **🎨 디자인 파트 (Designer)** | 2단계: 시각화 작업 시 | [ ] 모든 핵심 CTA 버튼의 내부 텍스트는 Pretendard Bold (700), 16px로 통일되었는지 확인. <br> [ ] 섹션 제목(H2)과 본문 간 충분한 여백(Padding) 확보를 통해 권위적/학술적인 느낌을 강조했는지 점검. <br> [ ] 그래프나 데이터 시각화에 사용되는 축 레이블은 'Caption' (12~14px) 크기를 준수하는지 확인. | | |
| **💻 개발/기술 파트 (Developer)** | 3단계: 웹 구현 시 | [ ] 모든 인터페이스 요소(버튼, 메뉴 등)의 기본 폰트를 Pretendard로 설정했는지 CSS 검증. <br> [ ] 'tel:' 링크를 포함한 모든 연락처 정보가 표준화된 형태로 통합되었는지 확인 (02-6494-0119). <br> [ ] 보고서 다운로드 시, PDF Export의 기본 폰트 설정을 Pretendard로 강제하는지 기술 스펙 검토. | | |
| **📢 공통/검토 파트 (Reviewer)** | 최종 점검 단계 | [ ] 문서 최상단에 "표준 서체: Pretendard" 배너 또는 고정 텍스트가 명시되었는지 확인. <br> [ ] 모든 내용이 계층 구조(H1-Body-Caption)를 따르며, 정보의 위계가 명확한지 교차 검증. | | |
